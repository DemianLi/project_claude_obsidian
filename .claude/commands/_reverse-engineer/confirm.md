# Phase 1：輸入確認

詢問使用者：

```
① 這次要分析哪些檔案？（指定路徑 或 貼上內容）
② 目標是什麼？
   (A) 了解整體架構
   (B) 找出特定功能的邏輯
   (C) 準備擴充某個模組
③ 甲方說過有什麼已知的業務邏輯嗎？（作為推測的對照基準）
```

確認輸入類型：

| 旗標 | 重點 |
|------|------|
| `--db [schema]` | 優先執行「資料庫分析」部分 |
| `--api [路由]` | 優先執行「API 路由分析」部分 |
| `--full` | 所有層次全面掃描（適合初次接案） |
| 無旗標 | 依輸入內容自動判斷分析重點 |

確認完成後：
```
📋 分析計畫確認
目標：{分析目標}
輸入：{檔案/schema 摘要}
重點：{DB / API / 全面}

已知業務邏輯（對照基準）：
  {使用者提供的，或「無」}

開始分析？(Y/n)
```

→ 確認後讀取 `_reverse-engineer/extract.md`
