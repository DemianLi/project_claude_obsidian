# Phase 3：產生報告

## 對外格式（甲方導向）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[專案名稱] 進度報告
報告期間：{起} ～ {迄}
製作日期：{YYYY-MM-DD}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

一、本期完成項目
- ✅ {功能名稱}：{一句話說明，避免技術術語}

二、進行中項目
- 🔄 {功能名稱}：預計完成時間 {日期}，目前完成 {N}%

三、下期計畫
- {功能名稱}：預計 {日期} 開始/{完成}

四、需要甲方協助（重要）

以下項目需要貴方協助確認，若未能在 {日期} 前收到回覆，可能影響後續進度：
{若無：「本期無需甲方協助的項目。」}

① {問題描述}
   → 需要確認：{具體問題}
   → 影響功能：{若延誤，哪些功能會被 block}
   → 期望回覆日期：{YYYY-MM-DD}

五、本期確認的追加需求
{若有 CR：列出本期核准的 CR 清單}
{若無：「本期無追加需求。」}

六、備注
{其他需要告知甲方的事項}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 內部格式（--internal）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[專案名稱] 內部進度報告｜{起} ～ {迄}

需求完成度：合約內 {N}/{Total}（{%}）｜追加 CR {N}/{Total}

本期完成：REQ-FXXX — {需求名稱}（{實際耗時} h）
進行中：  REQ-FXXX — {需求名稱}（預計還需 {N} h）

⚠️ 待解決：
  PENDING-XXX：等待甲方回覆 {N} 天
  INF-XXX：影響 REQ-FXXX 的推測仍未驗證

技術債：本期新增 {N} 條 / 積累 {N} 條
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

→ 完成後讀取 `_report/output.md`
