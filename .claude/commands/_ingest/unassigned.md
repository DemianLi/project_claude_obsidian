# 無主暫存流程（使用者無法確認專案歸屬時）

---

## 執行步驟

1. 解析文件，提取所有可識別的需求（不標記清晰度，只做初步整理）

2. 寫入 `_inbox/_unassigned/{文件名稱}-{YYYY-MM-DD}.md`：

```markdown
---
source: {文件名稱}
ingested: {YYYY-MM-DD}
project: 待定
---

# 無主需求暫存 — {文件名稱}

> 此文件尚未歸屬至任何專案。
> 確認後執行：`/ingest _inbox/_unassigned/{此檔案名}` 重新處理。

## 摘要
{2–3 句描述文件內容}

## 識別到的需求項目

### [U-001] {需求描述}
**原文**：{原始文字}
**初步分類**：功能需求 / 非功能需求 / 背景資訊 / 不確定

### [U-002] ...
```

3. 完成回報：
```
📥 已暫存至 _inbox/_unassigned/{文件名稱}-{日期}.md
識別出 {N} 個需求項目，等待確認專案後再正式整理。
確認後執行：/ingest _inbox/_unassigned/{文件名稱}-{日期}.md
```

> 無主暫存**不進入** Phase 2 的標準確認流程。
