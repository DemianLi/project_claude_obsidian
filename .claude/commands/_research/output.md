# Phase 5：呈現結果 + 後續行動

## 結果格式

```
🔍 Research 完成｜主題：{問題}｜專案：{名稱}

━━━ 約束輪廓 ━━━
環境：{摘要}
關鍵限制：{最重要的 2-3 條}

━━━ 方案排名（依相容性） ━━━

🥇 推薦：{方案 A}
   相容度：✅ 完全相容
   理由：{為什麼最適合這個專案}
   參考：{URL}

🥈 備選：{方案 B}
   相容度：🟡 需要確認 {N} 個問題
   問題清單：
     - {待確認問題}

🥉 不建議：{方案 C}
   相容度：❌ 不相容
   原因：{具體原因}

━━━ Claude 的理解確認 ━━━
「我理解{術語/業務規則}的意思是{Claude 的詮釋}，
 所以選擇方案 A 能夠滿足{相關需求}。
 這個理解正確嗎？」

━━━ 建議後續行動 ━━━
1. 確認以上 Claude 的理解
2. 回答待確認問題後，方案 B 可能升為首選
3. 若採用方案 A → 建議建立 ADR 記錄選型原因
```

## 使用者確認採用後

若使用者確認採用某方案：
1. 詢問是否建立 ADR → `projects/{專案}/04-decisions/ADR-XXX.md`
2. 若搜索過程中發現新的技術限制 → 更新 `existing-system.md`
3. 若發現新的領域理解 → 更新 `domain-knowledge.md`
4. Append `wiki/log.md` 記錄此次研究
