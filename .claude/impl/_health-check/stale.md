# Phase 3：孤立與過期項目掃描

## 3-1. _pending.md 項目年齡分析
```
對每條 PENDING-XXX：
  - 計算「最後更新日期」距今天數
  - ≥ 30 天 → 標記 ⚠️ 逾期
  - ≥ 60 天 → 標記 🔴 嚴重逾期（建議主動追問甲方）
  - 「阻塞性：是」且逾期 → 最高優先警示
```

## 3-2. _inferred.md 高風險推測
```
對每條 INF-XXX / GAP-XXX：
  - 信心度 🔴 低 → 標記為高風險
  - 超過 30 天未驗證 → 標記逾期
  - 有對應 REQ 被實作中 → 建議升格或廢棄推測
```

## 3-3. ADR 孤立檢查
```
對每份 ADR：
  - 若 affects_reqs 非空 → 確認對應 REQ 仍存在（未被廢棄）
  - 若 status: accepted 但對應 REQ 為 ❌ 已廢棄 → 標記不一致
```

## 3-4. Bug 老化掃描
```
對 06-qa-testing/bugs.md 中每條 BUG：
  - 未關閉且超過 14 天 → 標記 ⚠️
  - 未關閉且「優先級：🔴 高」超過 7 天 → 標記 🔴
```

輸出格式：
```
⏰ 逾期待釐清（{N} 條）：
  {專案} PENDING-{ID}：{描述}（已逾期 {N} 天）{阻塞：⚠️}

🔴 高風險推測（{N} 條）：
  {專案} INF-{ID}：信心度低 + {N} 天未驗證

🐛 老化 Bug（{N} 個）：
  {專案} BUG-{ID}：{N} 天未關閉，優先級 {🔴/🟡}
```

→ 完成後讀取 `_health-check/knowledge.md`（若完整診斷模式）
