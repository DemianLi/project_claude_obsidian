# Phase 4：執行清理（人工確認後）+ 完成回報

依使用者的選擇執行以下操作：

## 標記完全解決
在條目加上解決標記，不刪除（保留歷史紀錄）：
```markdown
### ~~PENDING-003：「{描述}」~~
**狀態**：✅ 已解決（{YYYY-MM-DD}）
**解決方式**：由 REQ-FXXX、REQ-FXXX 涵蓋
```

## 拆分部分解決的條目
```markdown
### ~~PENDING-007：「{描述}」~~（部分解決）
**狀態**：🔶 部分解決（{YYYY-MM-DD}）
**已解決**：{已解決部分}，見 REQ-FXXX
**剩餘問題**：→ 見 PENDING-016（繼承自本條）

### PENDING-016：{剩餘問題標題}（繼承自 PENDING-007）
**問題**：{剩餘未解決的疑問}
```

## 合併內部重複
保留較完整的條目，另一條標記「已合併至 PENDING-XXX」。

---

## 完成回報

```
✅ 對帳完成｜專案：{名稱}

清理結果：
  標記完全解決：{N} 條
  拆分部分解決：{N} 條 → 新增 {N} 條繼承項目
  合併重複：{N} 組
  仍待釐清：{N} 條

知識庫健康度：
  _pending.md：{N} 條（已解決 {N} / 未解決 {N}）
  _inferred.md：{N} 條推測 + {N} 個缺口

📌 剩餘待辦：
  - 超過 30 天未釐清：{N} 條（建議主動追問甲方）
  - 邏輯缺口仍開放：{N} 個（擴充相關功能前必須解決）
```

append 記錄至 `wiki/log.md`。
