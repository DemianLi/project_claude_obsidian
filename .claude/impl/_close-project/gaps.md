# Phase 1：收尾確認（防止遺漏）

讀取以下檔案，生成完成度報告：
```
1. {專案}/01-requirements/functional.md → 未完成的需求
2. {專案}/06-qa-testing/bugs.md        → 未關閉的 bug
3. {專案}/01-requirements/_pending.md  → 未解決的待釐清項目
4. {專案}/01-requirements/_inferred.md → 未驗證的高風險推測
```

**等待使用者確認後才繼續。**

```
🔍 專案收尾前置確認｜{專案名稱}

━━━ ⚠️ 以下項目尚未完成 ━━━

📋 未完成需求（{N} 條）
  - REQ-F{ID}：{需求名稱}（狀態：{狀態}）

🐛 未關閉 Bug（{N} 個）
  - BUG-{ID}：{描述}（優先級：{🔴/🟡/🟢}）

❓ 未解決的待釐清項目（{N} 條）
  - PENDING-{ID}：{描述}（影響：{有無影響驗收}）

━━━ 建議行動 ━━━
選項 A：先處理以上問題，再執行 /close-project
選項 B：確認這些是「已知遺留問題」，記錄後繼續封存
        （適合：合約外的問題、甲方知情並接受的 known issue）

如何繼續？(A / B / 取消)
```

若選 B：使用者說明每個遺留問題的處置方式（記入 CHANGELOG「已知問題」）。

若無任何未完成項目：
```
✅ 所有需求已完成、Bug 已關閉、待釐清項目已解決。繼續執行 Phase 2。
```

→ 完成後讀取 `_close-project/snapshot.md`
