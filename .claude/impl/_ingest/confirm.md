# Phase 2：呈現分析報告（等待人工確認）

**在使用者確認前，不寫入任何檔案。**

---

## 報告格式

```
📋 解析報告｜專案：{名稱}｜來源：{文件名稱}

━━━ 🏭 背景知識（{N} 條）━━━
[0-A] 關係人：{...} → 寫入 stakeholders.md？ (Y/n)
[0-B] 現有系統：{...} → 寫入 existing-system.md？ (Y/n)
[0-C] 領域背景：{...} → 寫入 domain-knowledge.md？ (Y/n)

━━━ 🟢 明確需求（{N} 條）━━━
[1] {需求標題}
    驗收條件：{...}
    → 確認寫入？ (Y/n)

━━━ 🟡 模糊需求（{N} 條）━━━
[2] {需求標題}
    問題：{缺少什麼}  建議：{建議版本}
    → (A) 用建議版本  (B) 存入 _pending.md  (C) 自訂描述

━━━ 🔴 衝突項目（{N} 條）━━━
[3] {衝突描述}
    既有（REQ-FXXX）：{...}  新文件：{...}
    → (A) 以新文件為準  (B) 保留既有

━━━ 🔗 對帳：與現有 _pending / _inferred 的關係（{N} 筆）━━━
[P] PENDING-XXX：{描述}  ↕ 語意重疊：新文件 [{N}] 涵蓋了此問題
    → (A) 完全解決，標記 ✅  (B) 部分解決  (C) 無關

[I] INF-XXX：{推測描述}  ↕ 語意重疊：新文件 [{N}] 提到相關內容
    → (A) 確認推測 → 升格 REQ  (B) 未提及，繼續保留  (C) 否定 → 廢棄

━━━ 📊 統計 ━━━
明確：{N} ｜ 模糊：{N} ｜ 衝突：{N} ｜ 對帳：{N}
```

→ 使用者確認後前往 `_ingest/write.md`（Phase 3）
