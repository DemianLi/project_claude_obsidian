# Phase 0-B：結構化資料處理

**適用**：CSV / Excel 表格 / 貼上的需求表格

---

## B-1：偵測格式

```
偵測到結構化資料輸入（{CSV / 貼上表格}）。
欄位數：{N}  欄位名稱：{欄1} / {欄2} / ...  資料行數：{N}
```

## B-2：欄位映射（自動建議 + 人工確認）

```
━━━ 欄位映射建議 ━━━
  {來源欄位} →  REQ「需求名稱」  ✅
  {描述欄位} →  REQ「描述」      ✅
  {優先級}   →  優先級標記       ✅
  {驗收欄位} →  REQ「驗收條件」  ✅
  {未知欄位} →  (A)需求名稱 (B)描述 (C)驗收條件 (D)忽略

映射確認後繼續解析。
```

## B-3：逐行生成 REQ 預覽

使用者確認映射後，每行轉換為 REQ 格式：

```
━━━ 從表格解析出 {N} 條需求 ━━━
[1] {需求名稱}（第 2 行）
    描述：{...}  驗收：{...}  優先級：{高/中/低}
    清晰度：🟢 明確 / 🟡 模糊（{原因}）
[2] ...
```

> 結構化輸入 ≠ 需求清晰。表格中的模糊描述仍需進 `_pending.md`。

## B-4：來源記錄

```
**來源文件**：Excel/表格輸入（{YYYY-MM-DD}）— 第 {N} 行
```

→ 完成後前往 `_ingest/confirm.md`（Phase 2，直接跳過 parse.md）
