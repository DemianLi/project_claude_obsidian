# Phase 0-D：大型文件分批處理

**適用**：≥ 30 頁 / ≥ 80,000 字元（SRS、完整規格書、大型合約）

> 分批「解析」→ 批次間「累積」→ 全部完成後「整合確認一次」

---

## D-1：結構掃描（只掃結構，不深讀內容）

```
📄 偵測到大型文件：{名稱}  估計規模：{N} 頁 / {約 N 萬字}

識別章節：
  第 1 章：{名稱}（第 1–15 頁）
  第 2 章：{名稱}（第 16–40 頁）... （依實際章節列出）

初步判斷：功能需求章節 / 非功能需求 / 背景 / 附錄
```

## D-2：分批方案確認

```
建議分 {N} 批（每批 30–40 頁）：
  批次 1：第 1 章（第 1–15 頁）— {說明}
  批次 2：第 2 章（第 16–40 頁）— {說明}
  ...

選擇：(A) 依建議方案  (B) 自訂頁碼範圍  (C) 跳過附錄
```

## D-3：逐批解析（每批執行 Phase 1，完成後給快報）

```
━━━ 批次 {N}/{Total} 完成 ━━━ 來源：第 {X}–{Y} 頁
  功能需求：{N} 條（{M} 明確 / {K} 模糊）｜非功能：{N}｜衝突：{N}
  累積進度：已處理 {N}/{Total} 批，共 {K} 條

快速預覽（前 5 條）：
  [F-001] {標題}（🟢 / 🟡）
  ...

(Y) 繼續下一批  (N) 本批有誤，重新解析  (P) 看完整本批結果
```

## D-4：跨批去重（全批完成後）

```
發現可能重複：
  批次 2 [F-012] 「使用者可以搜尋訂單」
  批次 3 [F-031] 「訂單查詢，支援多條件篩選」
  → (A) 合併（取較詳細的）  (B) 保留兩條
```

## D-5：整合確認（等同 Phase 2，按章節分組）

進入 `_ingest/confirm.md`，但報告按章節分組呈現。

## D-6：寫入 + 索引建立（額外步驟，在 Phase 3 後執行）

1. 自動建立/更新 `functional-index.md`（各模組 REQ 數量）
2. 各 `functional/{module}.md` 頂部 REQ 快速掃描表自動填入
3. 完成回報加入分批統計：批數 / 去重合併數 / 各模組寫入數
