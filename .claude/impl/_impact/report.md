# Phase 4：輸出影響報告

## 呈現格式

```
📊 影響範疇分析｜{需求/CR 描述}

━━━ 資料層 ━━━
{Phase 2-1 結果}

━━━ API 層 ━━━
{Phase 2-2 結果}

━━━ 既有需求衝突 ━━━
{Phase 2-3 結果}

━━━ 技術決策衝突 ━━━
{Phase 2-4 結果}

━━━ 複雜度估算 ━━━
{Phase 3 結果}

━━━ 建議行動 ━━━
1. {根據分析給出建議，例如：}
   REQ-F003 有衝突 → 建議先執行 /query REQ-F003 確認處理方式
   ADR-003 可能需修訂 → 建議先討論再實作
   資料遷移涉及大量資料 → 建議在非尖峰時段執行
```

## 若搭配 /cr 使用

將分析結果寫入 `04-decisions/CR-{N}-*.md` 的「影響範疇分析」區塊：
- 填入影響的現有需求表格
- 填入預估工時和複雜度
