# Phase 4：健康檢查 + 生成改善報告

## 4-A. 快速健康檢查

```
- 「只有 overview 但沒有需求文件」的專案 → 提醒補充
- 「有需求但沒有 ADR」的重要決策 → 建議補記
- wiki/index.md 是否需要更新狀態
- tech-stack/ 中「記錄完整度：⬜ 空白」但已使用過的服務 → 提醒補充
```

## 4-B. 生成改善報告

```
🔄 Self-Improve 完成報告

掃描範圍：{N} 個專案，{N} 份 dev-notes
掃描期間：{起始日} → {結束日}

📈 本期提取
  - 新增模式：{N} 個
  - 第三方服務坑記錄：{N} 條（{服務名稱列表}）
  - 新增教訓：{N} 條
  - 更新甲方偏好：{N} 條

🏥 知識庫健康度
  - 完整度：{N}%（有 overview + requirements + architecture）
  - 需要補充的專案：{列表}

💡 Claude 的觀察
  {2-3 條高層次的洞察}
  例：「本季需求變更集中在 UI 層，建議未來在需求確認階段增加 Mockup 確認步驟」

🎯 建議下次實作時注意
  - {基於本期教訓的具體建議}
```

## 4-C. 更新 wiki（自動執行，不需使用者確認）

**wiki/log.md**：
```

- 新增模式：{N} / 教訓：{N} / 服務踩坑：{N}
```
