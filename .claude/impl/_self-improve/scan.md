# Phase 1：掃描 Dev Notes

讀取最近 30 天的 `05-dev-notes/` 檔案（或指定專案的）。

提取以下資訊：

```
掃描範圍：{N} 個專案，{N} 份 dev-notes（{起始日} → {結束日}）

提取結果：

重複出現的問題類型：
  - {問題類型}（出現 {N} 次）
  - {問題類型}（出現 {N} 次）

有效的解法模式（重複 3 次以上）：
  - {解法描述}（使用於 {專案}/{功能}）
  - {解法描述}

第三方服務相關段落：
  識別關鍵字：API、串接、Webhook、OAuth、金流、SMS、通知、
              憑證、Token、Rate limit、Sandbox、{服務商名稱}
  - [{服務名稱}] {dev-note 日期}：{一句話摘要}

甲方特有的偏好和限制：
  - {偏好描述}（{甲方名稱}）

架構決策的後果（好的和壞的）：
  - {決策描述} → {後果}
```

→ 完成後讀取 `_self-improve/patterns-tech.md`
