# Stage 3.5：協作衝突偵測

> 在 BEFORE CODING Stage 3 找出目標 REQ 後執行。
> 前提：`projects/{專案名稱}/.claude/collab.md` 存在。

---

## Step 1：讀取 collab/ 資料夾

讀取 `projects/{專案名稱}/.claude/collab/*.md`（每位工程師一個檔案），合併所有人的鎖定清單。

若資料夾不存在或為空 → 跳過此 Stage，繼續 Stage 4。

## Step 2：比對目標 REQ

將 Stage 3 識別出的目標 REQ ID（可能多個）與 `active_work` 中所有 `locked_reqs` 欄位比對。

**無衝突** → 繼續 Stage 4。

**有衝突** → 執行 Step 3。

## Step 3：衝突警告 + 要求確認

```
⚠️ 協作衝突偵測

以下 REQ 目前由其他工程師鎖定：

┌────────────┬──────────┬──────────┬────────────┐
│ REQ ID     │ 鎖定工程師│ 鎖定日期 │ 備注       │
├────────────┼──────────┼──────────┼────────────┤
│ {REQ-FXXX} │ {縮寫}   │ {日期}   │ {note}     │
└────────────┴──────────┴──────────┴────────────┘

繼續實作可能導致與 {縮寫} 的工作產生衝突。

是否仍要繼續？
  Y — 確認繼續（建議先與 {縮寫} 溝通）
  N — 停止，先與對方確認後再繼續
```

**使用者選 N** → 停止，顯示：
```
🛑 已停止。建議步驟：
  1. 聯繫 {工程師名稱}（{縮寫}），確認 {REQ-FXXX} 的實作進度
  2. 確認後再繼續；若對方已完成，請他在 /save 時解鎖
```

**使用者選 Y** → 繼續 Stage 4，並在 session 結束 `/save` 時提醒：
```
💡 本次跨越鎖定繼續實作了 {REQ-FXXX}（原鎖定人：{縮寫}），
   建議 /save 時更新 collab.md 鎖定狀態。
```
