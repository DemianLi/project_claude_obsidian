# /cr --reject 和 /cr --list

---

## /cr --reject CR-{N} — 拒絕變更請求

讀取 `04-decisions/CR-{N}-*.md`，執行：

> `scope.md` 是團隊共用整份文件（見 `wiki/collab-protocol.md` 三），異動前先確認本機是最新版本；若內容跟讀取時不同（疑似其他人剛異動過），停止並回報差異。

### Step 1：詢問拒絕原因
```
拒絕 CR-{N} 的原因？
(A) 超出本案技術範疇
(B) 甲方確認不做（預算/時程）
(C) 與已確認需求衝突
(D) 其他：{說明}
```

### Step 2：更新 scope.md 的超出範疇表
```markdown
| {功能描述} | {提出日期} | {提出人} | 超出範疇 | {拒絕原因} |
```

完成後 append `wiki/log.md`：「{日期} {縮寫} 異動了 scope.md：拒絕 CR-{N}，{功能描述} 列入超出範疇」

### Step 3：更新 CR 文件狀態
```markdown
status: ❌ 已拒絕
**決策結果**：❌ 拒絕，{原因}
```

### Step 4：回報
```
❌ CR-{N} 已拒絕並記錄
功能「{名稱}」已列入超出範疇清單（scope.md）。
若甲方後續重新提出，執行 /cr 再評估。
```

## /cr --list — 列出所有 CR

讀取 `04-decisions/CR-*.md`，彙整成表格：

```
📋 變更請求列表｜專案：{名稱}

| CR 編號 | 主題 | 提出日期 | 狀態 | 關聯需求 |
|---------|------|----------|------|----------|
| CR-001  | ...  | ...      | 🕐 評估中 | — |
| CR-002  | ...  | ...      | ✅ 已核准 | REQ-F010 |
| CR-003  | ...  | ...      | ❌ 已拒絕 | — |

共 {N} 條 CR：評估中 {N} / 已核准 {N} / 已拒絕 {N}
```
