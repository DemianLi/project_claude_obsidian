# 停止回報格式（當 BEFORE CODING 發現 🔍INF / ❓GAP 時使用）

從對應 `_inferred/{ID}.md` 讀取後，展開以下格式回報：

```
🚫 實作暫停｜原因：存在未解決的推測/缺口

本次任務「{任務名稱}」在實作前需要先解決以下問題：

━━━ 🔍 推測待驗證 ━━━

INF-XXX：{推測標題，即檔案標頭}
  推測依據：{來自哪段程式碼/欄位}
  信心度：🟢 高 / 🟡 中 / 🔴 低
  需驗證：{需要問甲方的問題}
  （若檔案已記錄）⚠️ 若推測錯誤，實作後果：{說明代價}

  【建議問甲方】
  > {取自「需驗證」欄位，整理為可直接複製的問題文字}

━━━ ❓ 邏輯缺口 ━━━

GAP-XXX：{缺口標題，即檔案標頭}
  類型：⚡ 狀態機缺口 / 🗃️ 資料孤島 / 🔀 矛盾邏輯 / 🔘 孤立程式碼 / 🔢 魔術數字
  現象：{不完整/矛盾之處}
  影響：{為何無法安全實作}
  （若檔案已記錄）若強行實作的風險：{可能造成的 bug 或資料問題}

  【建議問甲方】
  > {取自「建議問甲方」欄位}

━━━ 建議行動 ━━━
1. 將以上問題傳給甲方確認
2. 收到回覆後執行 /ingest [甲方回覆] 更新知識庫
3. 推測/缺口升格為確認需求後，再繼續本次任務

要我幫你把以上問題整理成一封可以直接寄出的 email 嗎？
```
