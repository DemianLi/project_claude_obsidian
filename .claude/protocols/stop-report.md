# 停止回報格式（當 BEFORE CODING 發現 🔍INF / ❓GAP 時使用）

從 `_inferred.md` 讀取後，展開以下格式回報：

```
🚫 實作暫停｜原因：存在未解決的推測/缺口

本次任務「{任務名稱}」在實作前需要先解決以下問題：

━━━ 🔍 推測待驗證 ━━━

INF-XXX：{推測標題}
  推測內容：{Claude 推斷的業務規則}
  推測依據：{來自哪段程式碼/欄位}
  信心度：🟡 中
  ⚠️ 若推測錯誤，實作後果：{說明代價}

  【建議問甲方】
  > {可直接複製給甲方的問題文字}

━━━ ❓ 邏輯缺口 ━━━

GAP-XXX：{缺口標題}
  缺口描述：{不完整/矛盾之處}
  影響本次任務：{為何無法安全實作}
  若強行實作的風險：{可能造成的 bug 或資料問題}

  【建議問甲方】
  > {可直接複製給甲方的問題文字}

━━━ 建議行動 ━━━
1. 將以上問題傳給甲方確認
2. 收到回覆後執行 /ingest [甲方回覆] 更新知識庫
3. 推測/缺口升格為確認需求後，再繼續本次任務

要我幫你把以上問題整理成一封可以直接寄出的 email 嗎？
```
