# Schema：INFERRED（推測 INF / 邏輯缺口 GAP）

> 這是所有指令的唯一格式參考。
> `/reverse-engineer` 寫入時、`/health-check` 驗證時，均以此為準。
> 此目錄下所有內容均未經甲方驗證，**禁止作為實作依據**。

---

## 完整格式 — 推測（INF）

這是 `/reverse-engineer`（`_reverse-engineer/infer.md`）實際寫入的格式，標頭即推測內容摘要，不需獨立欄位：

```markdown
INF-{N}：{業務邏輯描述}
  推測依據：{1-2 行關鍵程式碼引用}
  信心度：🟢 高 / 🟡 中 / 🔴 低
  需驗證：{需要問甲方的問題}
```

選用擴充欄位（手動深入記錄、或推測進入驗證流程後可補上；`/reverse-engineer` 預設不寫入，不可因缺少而判定不健康）：
```markdown
**若推測錯誤，實作後果**：{說明推測錯方向的代價}
**影響的功能模組**：{哪些功能依賴這條推測}
**狀態**：待驗證
**更新紀錄**：
— `待驗證` → （驗證後填入）
```

## 完整格式 — 邏輯缺口（GAP）

這是 `/reverse-engineer`（`_reverse-engineer/gaps.md`）實際寫入的格式：

```markdown
GAP-{N}：{缺口描述}
  類型：⚡ 狀態機缺口 / 🗃️ 資料孤島 / 🔀 矛盾邏輯 / 🔘 孤立程式碼 / 🔢 魔術數字
  現象：{程式碼中觀察到的不完整/矛盾情形}
  影響：{擴充哪些功能前必須先釐清}
  建議問甲方：「{具體問題文字}」
```

選用擴充欄位（同上，預設不寫入）：
```markdown
**若強行實作的風險**：{量化影響程度}
**狀態**：待釐清
**更新紀錄**：
— `待釐清` → （釐清後填入）
```

## 必填欄位

| 欄位 | 適用類型 | 必填 | 說明 |
|------|---------|------|------|
| 標頭 `INF-{N}：{描述}` / `GAP-{N}：{描述}` | 兩者 | ✅ | 描述本身即內容摘要 |
| 推測依據 | INF | ✅ | 具體程式碼/欄位/行為引用 |
| 信心度 | INF | ✅ | 見下方有效值 |
| 需驗證 | INF | ✅ | 需要問甲方的問題 |
| 類型 | GAP | ✅ | 見下方有效值 |
| 現象 | GAP | ✅ | 具體觀察到的現象 |
| 影響 | GAP | ✅ | 哪條 REQ/模組在等 |
| 建議問甲方 | GAP | ✅ | 至少 1 句可直接使用的問題 |
| 若推測錯誤實作後果／影響的功能模組（INF）、若強行實作的風險（GAP）、狀態、更新紀錄 | 兩者 | ⬜ 選填 | 屬擴充記錄，預設不寫入，不影響健康判定 |

## 有效信心度值（INF 專用）

`🟢 高` / `🟡 中` / `🔴 低`

## 有效類型值（GAP 專用）

`⚡ 狀態機缺口` / `🗃️ 資料孤島` / `🔀 矛盾邏輯` / `🔘 孤立程式碼` / `🔢 魔術數字`

## 有效狀態值（僅填寫擴充欄位時適用）

| 狀態值 | 適用類型 | 含義 |
|--------|---------|------|
| `待驗證` | INF | 尚未經甲方確認 |
| `待釐清` | GAP | 尚未經甲方釐清 |
| `✅ 已驗證（升格為：REQ-FXXX）` | 兩者 | 已確認，已移入正式需求 |

---

## ID 規則

- `INF-{N}`：推測，同專案內唯一
- `GAP-{N}`：邏輯缺口，同專案內唯一

---

## 健康檢查規則（供 /health-check 使用）

一條 INF 被視為「不健康」如果：
- 缺少推測依據／信心度／需驗證任一欄位
- 信心度值不在有效列表中
- 需驗證內容為空
- 標頭 ID 格式不是 `INF-{數字}`

一條 GAP 被視為「不健康」如果：
- 缺少類型／現象／影響／建議問甲方任一欄位
- 類型值不在有效列表中
- 標頭 ID 格式不是 `GAP-{數字}`

兩者共通：
- 同一檔案含超過一個 `INF-` 或 `GAP-` 標頭（違反一檔一條目，`_index.md` 不適用此項）

> 選用擴充欄位（狀態、更新紀錄等）缺少時**不算不健康**——`/reverse-engineer` 預設不寫入，只在手動深入記錄或進入驗證流程時才會出現。

額外風險判定（供 `/health-check --stale` 使用，非格式錯誤）：
- 信心度 🔴 低（INF） → 標記高風險
- 超過 30 天未驗證/未釐清（以檔案建立時間估算） → 標記逾期
