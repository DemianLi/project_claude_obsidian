# Bug 追蹤清單

> 此文件追蹤測試期間（含 UAT）發現的所有問題。
> 每筆 bug 必須連結到對應的需求和測試案例，以便判斷根因類型。
>
> **根因類型說明：**
> - 🔧 實作缺陷：程式行為與已確認需求不符（需修正程式）
> - 📋 需求誤解：開發方對需求的理解有誤（需釐清後修正）
> - 🆕 新需求：甲方的期望超出現有確認需求（需走 /cr 流程）
> - 🔍 環境問題：僅在特定環境出現，非程式本身的問題

## Bug 記錄格式

```markdown
### BUG-{編號}：{問題一句話描述}

**嚴重程度**：🔴 高 / 🟡 中 / 🟢 低
**狀態**：🔴 未修復 / 🟡 修復中 / 🔵 待驗證 / ✅ 已關閉 / ⏭️ 不修復
**根因類型**：🔧 實作缺陷 / 📋 需求誤解 / 🆕 新需求 / 🔍 環境問題

**發現時間**：{YYYY-MM-DD}
**發現方式**：開發自測 / UAT 甲方回饋 / 測試案例 TC-{ID} / 正式環境
**發現人**：{姓名/角色}

**關聯需求**：{REQ-FXXX} {需求名稱}
**關聯測試案例**：{TC-FXXX-YY}（若有）

**問題描述**：
{詳細描述問題現象，包含：在哪個功能、輸入什麼、得到什麼結果}

**重現步驟**：
1. {步驟一}
2. {步驟二}
3. 實際結果：{...}
4. 預期結果：{...}

**環境資訊**：
- 作業系統：{OS}
- 瀏覽器：{Browser + 版本}（若適用）
- 測試環境：開發 / 測試 / 正式

**截圖/附件**：{路徑或描述}

**修復方案**：（修復後填寫）
{說明修了哪裡、為什麼這樣修}

**修復日期**：—
**關閉日期**：—
**關閉備註**：{若「不修復」，說明原因}
```
