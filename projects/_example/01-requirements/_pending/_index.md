## 使用流程

1. `/ingest` 解析文件 → 模糊項目自動建立 `_pending/{ID}.md` 並加入上方目錄
2. 拿項目內的「建議問甲方」文字去問甲方
3. 收到答覆後，執行 `/ingest [甲方回覆]` 或手動更新對應 `functional/{module}.md`
4. 將此項目從目錄移除，或標記 ✅ 已解決（保留檔案供歷史參考）

---

## 個別檔案格式（建立 `_pending/{ID}.md` 時使用）

```markdown

**來源**：{文件名稱} 第 {X} 頁
**問題**：{為什麼無法作為需求，例如：缺乏量測指標、有歧義}

**優先級**：🔴 阻塞（影響目前 sprint）/ 🟡 重要（本週應確認）/ 🟢 一般（不急）
**阻塞性**：是 / 否（若「是」，說明：哪條 REQ 或哪個功能模組在等這個澄清）
**預期確認日**：{YYYY-MM-DD / 不確定}

> {可以直接複製給甲方的問題文字}

**狀態**：🕐 等待回覆 / 🔄 已發問等待中 / ✅ 已解決（移入 functional/{module}.md REQ-FXXX）/ ⛔ 已廢棄
**建立日期**：{YYYY-MM-DD}
**最後更新**：—
```
