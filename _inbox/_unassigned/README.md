# _unassigned — 無主需求暫存區

此資料夾由 `/ingest` 自動管理。

## 用途

當你收到一份文件但還不確定它屬於哪個專案時，`/ingest` 會把解析結果暫存在這裡，
而不是強迫你立刻做決定或讓模糊的歸屬污染某個專案。

## 檔案命名

`{文件名稱}-{YYYY-MM-DD}.md`

## 後續處理

確認專案後，執行：

```
/ingest _inbox/_unassigned/{檔案名稱}
```

Claude 會重新走完 Phase 2-3（確認 → 寫入），並在完成後自動刪除此暫存檔。

## 注意

- 這裡的內容**不會**被 Claude 在實作時引用
- 定期檢查是否有長期滯留的無主文件（建議每週一次）
